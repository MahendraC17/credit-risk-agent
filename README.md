I built a credit risk decision system that separates prediction, explanation, and decision-making into distinct layers. The model (LightGBM with calibration) produces a reliable probability of default, which serves as the base risk. SHAP is used to understand what influenced the model’s prediction, providing feature-level contributions such as income, loan amount, or home ownership. However, SHAP is not used directly for decisions because it reflects statistical correlations, not business policy.

Instead, I introduced a signal layer that converts raw data into a small set of interpretable and stable risk factors, such as debt-to-income ratio (DTI) and historical default behavior. These signals are intentionally limited to features that are financially meaningful and consistently explainable. The system then aggregates the model’s base risk with these signals using a non-linear function to produce a final risk score. This ensures that the model captures complex patterns while the decision layer remains controlled and interpretable.

Finally, a decision engine maps the final risk into actions such as approve, approve with conditions, or reject, based on calibrated thresholds. This design ensures that the system is not just predictive but decision-capable, with clear separation between model behavior (what influenced the prediction) and business logic (what should drive decisions). This makes the system more stable, explainable, and closer to real-world underwriting practices. So far I have only implemented the system of scoring and expplaining the key factors of rejection and approval. Later pushes should focus on more on mathematical stability with data backed threshold. All of these is planned to be encapsulated using FASTAPI and Streamlit using deliver the expplanation using an agent.
